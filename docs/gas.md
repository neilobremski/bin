# `gas` ‚Äî CLI for the GAS Bridge

`gas` is a thin Python (stdlib-only) client that POSTs JSON to a Google Apps Script web app ‚Äî the "GAS Bridge" ‚Äî exposing Gmail, Drive, Sheets, Calendar, Tasks, Contacts, Docs, and Gemini under a single hierarchical command surface.

```
gas <group> <command> [--flags ...]
```

Help is generated by `argparse` at every level: `gas --help`, `gas gmail --help`, `gas gmail send --help`.

## Auth

`gas` resolves credentials through `n0b secrets get`:

- `GAS_BRIDGE_URL` env var, otherwise `~/lib/gas-bridge-url.txt` (`n0b secrets get GAS_BRIDGE_URL`)
- `GAS_BRIDGE_KEY` env var, otherwise `~/lib/gas-bridge-key.txt` (`n0b secrets get GAS_BRIDGE_KEY`)

If either is missing the CLI exits with a helpful error pointing at both options.

## Long-text and binary input

Three forms are accepted on every long-text flag (`--body`, `--query`, `--prompt`, `--description`, `--text`, `--payload`, `--rows`, `--values`, `--headers`, ‚Ä¶):

- inline string ‚Äî `--body "hello"`
- `@/path/to/file` ‚Äî read the file
- `-` ‚Äî read stdin

Binary blobs (`--data-base64` on `drive upload` / `drive upsert`) accept `@/path/to/file` (read + base64-encode), `-` (stdin ‚Üí base64), or a pre-encoded base64 string.

## `--jq` pass-through

`--jq <expr>` is captured before argparse runs and applied to stdout JSON via the `jq` binary. The flag works at any position; if the expression is omitted it defaults to `.`.

```
gas info --jq '.email_remaining'
gas gmail search --query "is:unread" --jq '.messages | length'
```

## Output

JSON is written to stdout (pretty when stdout is a terminal, compact when piped). Status and errors go to stderr. A daily one-line audit log is appended to `~/.gas/logs/YYYY-MM-DD.txt`.

## Groups

### `gmail`

Send mail, search threads, fetch full threads, mark read, archive, reply, label/unlabel/star, list-or-save attachments. Drafts: `draft.create`, `draft.list`, `draft.get`, `draft.send`, `draft.delete`. The `gmail check` command is a bridge-level alias for `gmail search`.

```
gas gmail send --to friend@example.com --subject "hi" --body "test"
gas gmail search --query "is:unread from:boss@" --count 5
echo "long body" | gas gmail send --to friend@example.com --subject "hi" --body -
gas gmail attachments --thread-id <ID> --save-to-drive --folder-id <FID>
```

### `drive`

`list` / `create` (document, spreadsheet, folder, raw file) / `upload` / `upsert` (replace by name) / `download` (returns base64) / `delete` (trash) / `folders`.

```
gas drive list --query "title contains 'circ-'" --count 10
gas drive upload --name notes.txt --data-base64 @/tmp/notes.txt --folder-id <FID>
echo "hello" | gas drive upload --name greeting.txt --data-base64 -
```

### `sheets`

`read` (A1 range), `append` (rows as 2D JSON), `create` (new spreadsheet, optional headers), `update` (set values in a range).

### `calendar`

`list` upcoming events, `create` an event, `delete` by ID, `calendars` to list all calendars on the account.

### `tasks`

`list` task lists with their tasks, `create` a task, `update` an existing task. (Requires the Tasks advanced service to be enabled at the bridge.)

### `contacts`

`list` ‚Äî read-only. Returns name, first email, first phone per contact.

### `config`

`get [--key NAME]` / `set --set KEY=VALUE [--set ...]` ‚Äî reads/writes Script Properties on the bridge. Disabled by default at the bridge; enable from the Apps Script editor.

### `docs`

`create --title <T> [--body <text>]` ‚Äî creates a new Google Doc.

### `gemini`

`ask --prompt <text> [--system <text>] [--model gemini-2.0-flash] [--max-tokens 2048] [--temperature 0.7]`. Uses the bridge's API key (Script Property) or falls back to OAuth.

### `token`

`token get` ‚Äî returns the bridge's OAuth access token. Disabled by default at the bridge.

## Top-level actions

- `info` ‚Äî bridge metadata (account, version, available actions).
- `actions` ‚Äî list of actions the live bridge supports (calls `info` and filters to `.actions`). Use this when you want a quick capability check before running something.
- `call <action> [KEY=VALUE ...]` ‚Äî passthrough escape hatch. Invokes any action by name with arbitrary KEY=VALUE params, type-coerced like the prior CLI (JSON arrays/objects, ints, booleans, `@/path` for file contents, `-` for stdin). Use this for actions not curated as subcommands here, or for future bridge actions added after this CLI was built. Unknown actions fail loudly with the bridge's own `available` list.
- `quota` ‚Äî daily Gmail/Drive quotas plus today's per-service usage counters.
- `translate --text <text> [--from auto] [--to en]` ‚Äî `LanguageApp.translate`.
- `fetch --url <URL> [--method get] [--payload <body>] [--content-type <ct>] [--header K=V]` ‚Äî server-side `UrlFetchApp` (disabled by default at the bridge).

## Examples

```
gas info
gas actions                                         # list of action names the live bridge supports
gas quota --jq '.email_remaining'
gas gmail send --to me@example.com --subject "ping" --body "hello"
gas drive list --query "title contains 'meeting'"
gas drive upload --name screenshot.png --data-base64 @/tmp/shot.png
gas sheets read --name "Q3 Plan" --range "Sheet1!A1:D50"
gas gemini ask --prompt "Summarize Apps Script Properties Service in 3 bullets."

# call escape hatch ‚Äî useful when the bridge gains a new action
gas call gmail.search query="is:unread" count=5
gas call chat.send space="spaces/abc" message="hi"  # if not yet implemented, bridge replies with available[]
```
